COMMANDS
- kubectl cluster-info
- kubectl get nodes/pod/deployment/service
- kubectl get events (log)

CREATE DEPLOYMENT
- kubectl create deployment [deployment_name] --image=[image_name]

CREATE SERVICE
- to make a pod accessible from outside kubernetes, expose the pod as a service
- kubectl expose deployment podname --type=LoadBalancer --port=8080

CLEAN UP
- kubectl delete service/deployment [name]
- minikube stop
- minikube delete

TROUBLESHOOTING
- kubectl get [resource] : list resources
- kubectl describe [resource] : show detailed information about a resource
- kubectl logs [podname] : print the logs from a container in a pod
- kubectl exec [podname] : execute a command on a container in a pod
- kubectl exec -ti $POD_NAME -- bash : starts bash session in pod

SCALING
- kubectl get rs : show infos about replicas
- kubectl scale [deployment_name] --replicas=[number_of_replicas_wanted]

UPDATING
- kubectl set image [deployment_name] [new_image_name]
- kubectl rollout status : check update
- kubectl rollout undo [deployment_name] : roll back to last working version
